# ClinicFlow Documentation

This folder contains documentation for the ClinicFlow solution, including architecture, role-based user manuals, deployment guidance, and compliance notes.

## Included documents

- `solution-overview.md` - High-level system summary and architecture.
- `tech-stack.md` - Technology stack, runtime dependencies, and system components.
- `admin-manual.md` - Administration tasks, onboarding, and tenant management.
- `user-manual.md` - User experience guidance for Doctors and Nurses.
- `financial-manual.md` - Finance and reception workflows for receptionists and accountants.
- `deployment-guide.md` - How to install, configure, and deploy ClinicFlow.
- `security-and-compliance.md` - Data protection, privacy, and compliance considerations.
- `documentation-agent.md` - The documentation steward and daily review process.

## How to use these documents

1. Start with `solution-overview.md` to understand the overall architecture and purpose.
2. Read `tech-stack.md` to learn how the application is built and what dependencies are used.
3. Use the role manuals for operational guidance and user-facing process descriptions.
4. Follow `deployment-guide.md` to run and deploy the application safely.
5. Review `security-and-compliance.md` before production rollout.

6. Use `documentation-agent.md` to understand the daily documentation review workflow and automation.

> Note: Regulatory and compliance PDFs are available in `docs/regulation/` for governance and standards references.
